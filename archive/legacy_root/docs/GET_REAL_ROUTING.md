# 🚀 Getting Real OTP Routing Working

## The Issue
- OTP server is **not running** (that's why you see 404)
- Your chatbot falls back to DEMO_MODE
- You want **real Dublin transit routes**

## Solution: Start OTP Server

### Option 1: Quick Start (Recommended)
1. **Open CMD** (right-click Windows Start → Command Prompt)
2. **Copy-paste this entire block:**
```batch
cd /d E:\irish_transport_ai\otp\graphs\default
java -Xmx10G -jar E:\OpenTripPlanner\otp-shaded\target\otp-shaded-2.8.1.jar --load --serve .
```

### Option 2: Use Batch File
1. **Double-click:** `E:\irish_transport_ai\run_otp.cmd`
2. A CMD window will open and start OTP

---

## Wait for This Message
Once you see this, OTP is ready:
```
OTP UPDATERS INITIALIZED - OTP 2.8.1 is ready for routing!
Grizzly server running.
```

This takes **30-60 seconds**. You'll see:
- "Graph loaded"
- "Transit loaded"  
- "Linking transit stops"
- Then "ready for routing!"

---

## Keep This Window Open
**Important:** OTP must stay running while you use the chatbot.

You'll have **2 windows open:**
1. **CMD Window** - OTP server (keep running)
2. **PowerShell Window** - Chatbot (`streamlit run dashboard/chat.py`)

---

## Test OTP is Working

Once you see "ready", test in another PowerShell window:
```powershell
python -c "
import requests
r = requests.get('http://localhost:8080/routers/default/plan', 
    params={
        'fromPlace': '53.35,-6.25',
        'toPlace': '53.38,-6.27',
        'date': '20250627',
        'time': '14:00'
    }, timeout=5)
data = r.json()
routes = len(data.get('plan', {}).get('itineraries', []))
print(f'✓ OTP working! Found {routes} routes' if routes > 0 else '✗ Still no routes')
"
```

Expected: `✓ OTP working! Found 3 routes`

## Troubleshooting

### Port 8080 Already in Use?
```powershell
# Kill existing Java process
taskkill /IM java.exe /F
# Then restart OTP
```

### OTP Crashes?
- Check you have 10GB free RAM
- Close other apps
- Try again

### Still Getting 404?
- Wait for full startup (watch for "ready" message)
- Make sure CMD window is still open (OTP quit if window closed)
- Test with the test command above
